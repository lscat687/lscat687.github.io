# signed和unsigned区别

- unsigned无符号

以16位为例，无符号的整形取值范围：0～65535

- signed有符号

int类型默认有符号，二进制最高位为符号位，0为正，1为负

以16位为例，有符号的整形取值范围：-32768～32767 【-2^15～2^15-1】

- 负数在计算机中以补码的形式存储

    > 正数的原码=反码=补码
    
    > 负数的原码：1000 1000 （-8）
    
    > 负数的反码：符号位不变，其他位取反  1111 0111
    
    > 负数的补码：反码+1  1111 1000 

- 特殊情况 1000 0000  （-128） 0000 0000  （0）
- 示例 （假设short int是8位）
```
#include <stdio.h>
main()
{
   short int a=127,b,c,d;//定义无符号类型。
   b=a+1;
   c=a+2;
   d=a+3;
   printf("a=%d\nb=%d\nc=%d\nd=%d\n",a,b,c,d);
}
```

a的原码（a为正数，补码相同） 0111 1111

a+1 补码 1000 0000  即-128 （-128并没有原码和反码表示）

a+2 补码 1000 0001 反码（补码-1）1000 0000 原码 1111 1111 （-127）
